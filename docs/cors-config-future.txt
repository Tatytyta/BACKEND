# Configuración de CORS más específica (para usar en el futuro)

Cuando estés listo para pasar a producción, puedes modificar la configuración CORS en tu backend para ser más específica y segura. Aquí te muestro un ejemplo de cómo podrías modificar el archivo `main.ts`:

```typescript
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  const port = process.env.PORT || 3000;
  
  // Configuración CORS más específica
  app.enableCors({
    origin: [
      'https://tu-frontend-produccion.onrender.com', // URL de tu frontend en producción
      'http://localhost:5173',                       // URL de tu frontend en desarrollo
    ],
    methods: ['GET', 'POST', 'PUT', 'DELETE', 'PATCH'],
    credentials: true,
  });
  
  await app.listen(port);

  console.log(`Servidor escuchando en el puerto ${port}`);
}
bootstrap();
```

Esta configuración limitará las peticiones CORS solo a los orígenes especificados, lo que aumenta la seguridad de tu aplicación.
